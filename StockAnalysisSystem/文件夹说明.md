# 项目文件夹说明

## 📁 .vs 文件夹

### 作用
**Visual Studio 的本地配置文件**

- 存储 Visual Studio 的**用户特定设置**
- 包含编辑器窗口位置、断点、调试配置等
- 每个开发者的配置可能不同

### 内容
- `*.suo` - 解决方案用户选项
- 编辑器状态、调试信息等

### 是否可以删除？
✅ **可以删除**
- 不影响项目功能
- 删除后，Visual Studio 会重新生成
- 已在 `.gitignore` 中，不会被提交到代码仓库

---

## 📁 .vscode 文件夹

### 作用
**VS Code / Cursor 的配置文件**

包含编辑器和工作区的配置：

- `launch.json` - **调试配置**（非常重要）
  - 定义如何启动和调试程序
  - 例如：按 F5 运行项目的配置
  
- `tasks.json` - **任务配置**（重要）
  - 定义构建、还原等任务
  - 例如：Ctrl+Shift+B 构建项目的配置

- `settings.json` - **工作区设置**（可选）
  - 项目特定的编辑器设置

### 是否可以删除？
⚠️ **建议保留**
- 删除后，按 F5 调试功能可能无法使用
- 需要手动重新配置
- 通常**应该提交到代码仓库**（与其他开发者共享配置）

**如果删除**：
- 仍然可以通过终端运行 `dotnet run`
- 但会失去便捷的调试功能

---

## 📁 bin 文件夹

### 作用
**编译输出文件夹**

包含项目**编译生成的文件**：

- `StockAnalysisSystem.exe` - 可执行文件
- `StockAnalysisSystem.dll` - 程序集文件
- 依赖的 DLL 文件（NuGet 包）
- 配置文件等

### 目录结构
```
bin/
  └── Debug/           # Debug 模式输出
      └── net8.0-windows/
          ├── StockAnalysisSystem.exe
          ├── StockAnalysisSystem.dll
          └── ...
  └── Release/         # Release 模式输出（如果有）
      └── ...
```

### 是否可以删除？
✅ **可以随时删除**
- 这些是**自动生成**的文件
- 运行 `dotnet build` 会重新生成
- **已在 `.gitignore` 中**，不会被提交到代码仓库

**何时需要删除**：
- 清理项目时
- 解决构建问题时
- 减少项目体积时

**删除后如何恢复**：
```bash
dotnet build
```

---

## 📁 obj 文件夹

### 作用
**编译中间文件**

包含编译过程中的临时文件：

- 生成的代码文件（`.g.cs`）
- 编译缓存
- 程序集引用信息

### 是否可以删除？
✅ **可以随时删除**
- 编译过程产生的**临时文件**
- 运行 `dotnet build` 会重新生成
- **已在 `.gitignore` 中**

---

## 📋 总结对比

| 文件夹 | 作用 | 是否可删除 | 是否提交到 Git |
|--------|------|-----------|---------------|
| **.vs** | Visual Studio 用户设置 | ✅ 可以 | ❌ 不提交 |
| **.vscode** | VS Code 配置 | ⚠️ 建议保留 | ✅ 应该提交 |
| **bin** | 编译输出 | ✅ 可以 | ❌ 不提交 |
| **obj** | 编译临时文件 | ✅ 可以 | ❌ 不提交 |

---

## 💡 建议

### 保留的文件/文件夹
- ✅ `.vscode/` - 保留，方便调试和开发
- ✅ 源代码文件
- ✅ 配置文件（如 `appsettings.json`）

### 可以删除的文件/文件夹
- ✅ `bin/` - 可以删除，节省空间
- ✅ `obj/` - 可以删除，节省空间
- ✅ `.vs/` - 如果使用 VS Code，可以删除

### 清理项目命令

```bash
# 清理构建输出
dotnet clean

# 或手动删除
Remove-Item -Path "bin" -Recurse -Force
Remove-Item -Path "obj" -Recurse -Force
Remove-Item -Path ".vs" -Recurse -Force
```

## 🔍 当前项目状态

查看你的项目大小：

```bash
# 查看各文件夹大小（PowerShell）
Get-ChildItem -Directory | ForEach-Object {
    $size = (Get-ChildItem $_.FullName -Recurse -ErrorAction SilentlyContinue | 
             Measure-Object -Property Length -Sum).Sum / 1MB
    [PSCustomObject]@{
        Folder = $_.Name
        SizeMB = [math]::Round($size, 2)
    }
} | Sort-Object SizeMB -Descending
```

**典型的文件夹大小**：
- `bin/` - 通常 10-50 MB
- `obj/` - 通常 1-5 MB
- `.vs/` - 通常 1-10 MB
- `.vscode/` - 通常 < 1 KB

---

**提示**：如果项目文件夹很大，通常是因为 `bin/` 和 `obj/` 文件夹。可以安全删除它们来节省空间。
